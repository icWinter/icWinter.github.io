spring 框架

spring 框架后端开发小计

1.接口返回数据

    对于java而言，任何数据格式的返回都以来一个java对象。如DO,DTO等对于数据库对应的java对象进行扩充。但实际上很难避免把不必要的字段进行返回，当然了，如果一个接口对应一个DTO,能很好避免这个问题。可实际上，很少有人这么用。

    既然每个接口都有特定的返回字段与格式，那么为什么不将返回字段定义到Controller上，从根本上解决这个问题。于是我首先想到使用自定义注解，用于Controller的方法上，每个接口都定义一组。然后在Spring写回Response 前进行拦截，获取栈中Controller的方法注解，获取待实际待返回字段，进行复制然后转换为JSON直接输出。

  . 自定义注解     必须是一组包含关系的

  . 处理拦截     查询资料发现Spring对开发者非常友好，预留了一个接口ResponseBodyAdvice ，需要实现2个接口方法即可。

  . 数据转换问题

    首先组合自定义的注解结构，使其和返回数据格式相互匹配。拼凑成JSONObject ，值可以为空。然后使用类似BeanUtils.copy(Object o1,Object o2);使用测试后，发现不可行，遂放弃。

    因此只能递归校验原返回值，和待返回字段进行匹配，不包含字段则直接删除。最后将JSONObject转换为javaObject 传递给返回包装类Result。前提是使用配置jackSon 不将空字段返回（具体配置为：在model的父类上加注解@JsonInclude(JsonInclude.Include.NON_EMPTY)）。

    而另外一个问题是，对于复杂对象的组合拼接，倒不是件容易的事情。例如对象本身就是树结构，双向链表等等，问题虽然比较麻烦。但解决方案仍会有的，已经在路上啦……

java 小记

1.对象池

    对象池化技术，使用空间换时间策略。

    尽可能减少对象的创建与回收时间，以达到优化目的。自定义对象池，对外提供借出、归还接口。动态维护一个对象集合，当程序申请调用时候，先判断池中是否有可用对象。如果没有，则加锁阻塞线程，直到有线程归还该类型对象后，再次借出。

    每次归还对象时候，都必须清空对象数据，防止数据污染。

2.并发编程

  • 1.CountDownLatch

        CountDownLatch适用于主线程启动多线程任务执行，阻塞主线程执行。直到所有子线程任务执行完毕，减法计数器为0，主线程恢复执行。

        CountDownLatch在初始化时候，必须指定多线程的数量，然后在每个子线程执行完毕中，调用减法计数。CountDownLatch，是不可复用的。当一个多任务执行完毕即失效。

  • 2.CyclicBarrier

        CyclicBarrier适用于多线程任务预处理，当启动多线程执行的任务时刻不固定。但需要这些多任务统一到达某个节点后，才能继续往下执行。（eg:短跑比赛中的预热环节，所有运动员，前期准备时间等都不固定。但是开始比赛的条件是:当所有运动员都在起跑线准备就绪，然后才能开始比赛。）

        CyclicBarrier阻塞所有子线程任务，当所有线程到达指定节点，然后统一放行。不同于CountDownLatch的是，CyclicBarrier支持复用reset。

  • 3.Semaphore

        Semaphore 信号量，则适用于限制某个代码块可以并发执行线程数的控制。如限制某个方法，同时只能有20个线程执行，其他试图执行该方法的线程，会进入阻塞状态，当达到限制上限时。

git 基本命令

  1. 创建远程仓库
  2. 初始化本地git 仓库, CMD: git init
  3. 添加项目文件到版本库, CMD: git add . or src or something else…
  4. 设置邮箱, CMD ： git config – user.email “123456798@git.com“
  5. 设置密码, CMD : git config – user.name “your name”
  6. 提交修改, CMD : git commit -m “初始化项目”
  7. 关联远程仓库，CMD : git remote add origin https://github.com/oraclexing/xxxxxx.git
  8. 拉取远程代码， CMD : git pull
  9. 关联远程分支，CMD : git branch –set-upstream-to=origin/master
  10. 推送代码到远程仓库，CMD : git push
